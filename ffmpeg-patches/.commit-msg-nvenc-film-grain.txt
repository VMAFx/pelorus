feat(nvenc): carry the Pelorus AV1 film-grain estimate into av1_nvenc

Stock nvenc.c never enables NVENC's AV1 film-grain synthesis, so a grain
estimate produced upstream is dropped and the grain is coded as residual. This
adds a `pelorus_film_grain` AVOption (default off, registered on av1_nvenc
only) that, when set, carries the AV1 (AOM) film-grain-synthesis parameters a
vf_pelorus_grain_estimate_vulkan stage emits into NVENC's hardware AV1
film-grain config so the decoder re-synthesizes the grain — the HW-AV1 leg of
the FGS round-trip (the H.274 BSF, patch 0010, covers HEVC; the AV1 software
encoders consume the native side data directly).

At init, when enabled, we set NV_ENC_CONFIG_AV1::enableFilmGrainParams and
point filmGrainParams at a persistent NV_ENC_FILM_GRAIN_PARAMS_AV1 in the
encoder context (zero-initialised => applyGrain 0, a no-op until a frame
carries an estimate). Per frame we refill that struct from the frame's grain
estimate and raise the AV1 pic-params filmGrainParamsUpdate flag (time-varying
model; a frame with no estimate keeps the last params). The estimate is read
from the native AV_FRAME_DATA_FILM_GRAIN_PARAMS channel (AV_FILM_GRAIN_PARAMS_AV1,
preferred — the canonical public field-for-field path) and, failing that, from
the PEL_SEC_FILMGRAIN interop section parsed inline (mirroring the
PEL_SEC_MOTION reader in the ME-hints patch). The AVFilmGrainAOMParams set maps
onto NVENC's struct directly: numbers shifted to its "minus N" / "plus 128"
bitfield conventions, point and AR-coefficient arrays copied element-wise, all
writes clamped to the field widths.

Generality: default off (zero behaviour change); compile-gated by a new
NVENC_HAVE_AV1_FILM_GRAIN macro (SDK 12.0+, where the AV1 encoder and the
film-grain struct landed) with a one-shot warn + graceful degrade when
unavailable; registered on av1_nvenc only (H.264/HEVC NVENC have no AV1 film
grain). The Pelorus blob is parsed inline (no libavcodec->libpelorus link for a
default-off feature) against the frozen append-only interop ABI (R1..R6). No
device/box assumptions.

No grain-match or BD-rate number ships here: the estimate is wired into NVENC's
AV1 film-grain config and verified to compile against the ffnvcodec headers,
register on av1_nvenc only, and replay onto pristine n8.1.1; the on-hardware
visual/BD-rate proof is a documented follow-up (ADR-0111 methodology).

If a frame carries no usable AV1 grain estimate the update flag stays clear, so
NVENC keeps the previous params (or the zero-init no-op), and behaviour is
unchanged.
