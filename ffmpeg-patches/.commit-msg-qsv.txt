feat(qsv): honor AV_FRAME_DATA_REGIONS_OF_INTEREST via the per-block QP map

Stock qsvenc.c already maps ROI side data onto mfxExtEncoderROI (rectangle
regions, capped at QSV_MAX_ROI_NUM and at rectangle granularity). This adds a
`pelorus_roi` AVOption (default off) that instead rasterizes each
AVRegionOfInterest's qoffset into Intel QSV's *dense* per-block delta-QP map
(mfxExtMBQP, Mode = MFX_MBQP_MODE_QP_DELTA, attached per frame via the existing
mfxEncodeCtrl ext-buffer chain; mfxExtCodingOption3::EnableMBQP turned on at
init), mirroring the merged NVENC qpDeltaMap patch (ADR-0114 Tier 1).

Dense map vs region rectangles: the vf_pelorus_analyze banding map is per-cell;
coalescing it into <=256 rectangles loses fidelity, whereas mfxExtMBQP carries a
per-16x16-block delta directly with no rectangle budget — same value path as the
NVENC delta-QP map it mirrors. qoffset->delta uses the libx264 convention
(qp_range = 51 + 6*(depth-8)); region precedence + self_size stride mirror
libx264's setup_roi.

Portable by construction (this ships to all users, not the dev box):
- block size is the SDK-documented 16x16 MBQP alignment for both AVC and HEVC
  (mfxExtMBQP), not any one GPU's coding-tree size; HEVC also carries BlockSize.
- runtime capability probe: EnableMBQP is only honored under CQP rate control
  (mfxExtCodingOption3 doc), so non-CQP sessions emit one warning and pass
  through unchanged; the whole path is compile-guarded by QSV_HAVE_MBQP
  (oneVPL / MediaSDK API >= 1.13) so older SDK headers no-op cleanly.
- no device-node / driver / entrypoint assumptions; zero behaviour change when
  the option is off (default).

Lets Pelorus's measured banding-ROI (vf_pelorus_analyze roi=1, ADR-0114 Tier 0)
steer Intel-hardware bit allocation through the same standard ROI side data the
merged analyze filter already emits — no new producer.

Verified: cc -fsyntax-only of the changed libavcodec TUs against the installed
oneVPL headers; full-stack regeneration applies cleanly. Full QSV ffmpeg rebuild
and on-Arc BD-rate proof are a follow-up (no fabricated numbers).
