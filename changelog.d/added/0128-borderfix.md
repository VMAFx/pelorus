- **`vf_pelorus_borderfix_vulkan`** (`ffmpeg-patches/files/vf_pelorus_borderfix_vulkan.c`): dirty-line / **border repair** ‚Äî a single-pass, zero-copy Vulkan compute (`FF_VK_REP_FLOAT` UNORM, bit-depth-agnostic; all planes by default) that is the GPU equivalent of FFmpeg's `fillborders=smear`. Cropped, telecined, and analog-captured sources carry garbage rows/columns at the frame edge (half-pixels, clamp lines, head-switching noise) that cost the encoder bits and read as a dirty border. Given the per-edge dirty-band widths, the filter clamps each pixel onto the clean interior rectangle `[left, w‚àí1‚àíright] √ó [top, h‚àí1‚àíbottom]` and stores the input sample read there ‚Äî smearing the nearest clean edge outward over the garbage band. The smear is exact and deterministic (no threshold, no blend); with all widths at the default `0` it is byte-identical pass-through. Runs entirely in VRAM, so a Vulkan pipeline never drops to a CPU `fillborders` and the `hwdownload`/`hwupload` round trip it would force. AVOptions: `left` (0), `right` (0), `top` (0), `bottom` (0) ‚Äî band widths per edge, in each plane's own pixels (so on 4:2:0 the chroma band is half the luma band) ‚Äî and `planes` (0xF = all). A pure pixel transform ‚Äî it does not link libpelorus, emits no interop side data (deps-only registration), and is the natural first stage (clean the edges before any later stage measures or processes the frame). Reference shader `libpelorus/shaders/pelorus_borderfix.comp` kept in lockstep. The transform is deterministic so no quality A/B is needed for correctness; a BD-rate note on a dirty-border corpus is a documented nice-to-have follow-up. Shipped as patch 0016. (ADR-0128)
