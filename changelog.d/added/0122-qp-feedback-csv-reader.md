- **Runnable QP-feedback reader** (closes ADR-0114 step 6 on a box without a working HW encoder): a SDK-free x265 `--csv --csv-log-level 2` stat reader populates the `PEL_SEC_QPREPORT` interop section with the encoder's *actual* post-encode numbers. `pel_x265_csv_parse` locates the `Type/POC/QP/Bits/PSNR` columns by header name and admits only coded-frame rows (dropping x265's trailing blank + `Summary` block); `pel_qp_report_from_x265_frames` folds the GOP into bit-weighted mean honored QP, total bits, bit-weighted PSNR, and a `honored_fraction` = sign-agreement between the requested and the honored per-frame delta-QP. Frame-granular (x265 CSV exposes no per-CTU QP), so `qp_valid` stays 0 and no per-cell map is written. This is **append-only consumer code** — no wire field/section changes, `PELORUS_ABI_MINOR` stays 1.1. A new demonstrator `tools/pelorus_qp_report` runs the full path (parse → fold → `pel_blob_pack` → `pel_blob_find_section` → print); a 17-frame `testsrc2` clip at `--qp 32 --aq-mode 0` yields **measured** `avg_qp ≈ 32.21`, `total_bits = 104744`, `psnr_y/u/v ≈ 38.6/37.6/37.0 dB`, `honored_fraction = 0.1765` (a flat request scores low because x265 spread the QP by slice type).
