- **MC→denoise motion-compensated warp (`vf_pelorus_denoise_vulkan mc=1`)**: the denoiser can now warp its temporal taps by an upstream `pelorus_mc` motion-vector field instead of averaging the same-coordinate sample, closing the high-motion ghosting gap that capped the GPU denoise win (ADR-0113, realized here). With `mc=1` the filter parses `PEL_SEC_MOTION` + the new `PEL_SEC_MOTION_CONF` from the input frame, uploads the quarter-pel MV grid and the per-block confidence grid as SSBOs (retained as exec dependencies so they survive an async submit), and replaces `pel_prev(t, pos)` with a bilinear-sampled `pel_prev_mc(t, pos + MV)` (nearest-cell MV, `mv_scale = 0.25`, chroma MV subsampled by `log2_chroma`). The warped tap is blended toward the same-coordinate tap by the per-cell confidence (`mix(same, warped, conf)`) so a noise-matched MV degrades gracefully, and the existing per-pixel `temporal_cut` gate still rejects bad/occluded taps. Pipeline order is load-bearing: `pelorus_mc_vulkan=meta=1,pelorus_denoise_vulkan=mc=1` (mc before denoise); with no upstream MV field denoise falls back to same-coordinate taps. Validated on a noisy ~2.5 px/frame scroll: the warp reaches Y-SSIM 0.5520 vs the clean reference where same-coordinate reaches 0.5478 (noisy input 0.5351) — active and beating same-coord on high-motion content. `pelorus_denoise.comp` updated in lockstep. ADR-0131.
- **`PEL_SEC_MOTION_CONF` interop section (ABI minor 2)**: a new append-only side-data section (`PelorusMotionConfSection`, 16 bytes) carrying a per-cell motion-confidence map (one `uint8`, `255*(1 − clamp(per-pixel winning SAD / 0.10))`, derived host-side from the SAD `vf_pelorus_mc` already reads back — no mc shader change). `PelorusMotionSection` stays frozen at 32 bytes; the conformance fixture round-trips the new section and asserts R3/R4 back-compat. The vmafx-vendored `interop.c` mirror is a documented follow-up (append-only keeps an older vmafx forward-compatible). ADR-0131.
