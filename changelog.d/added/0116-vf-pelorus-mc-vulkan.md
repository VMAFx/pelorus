- **`vf_pelorus_mc_vulkan`** (`ffmpeg-patches/files/vf_pelorus_mc_vulkan.c`): a GPU block-matching **motion estimator** that produces a per-block integer-pel motion-vector field for the current frame relative to the previous frame and attaches it as the pre-reserved `PEL_SEC_MOTION` interop section (frame scalars — global motion, magnitude mean/p95, entropy, scene-cut flag — plus the dense `int16 (dx,dy)` per-block grid). The frame passes through unchanged (the analyzer shape). One workgroup per block: cooperative SAD reduced in shared memory, then a predictor-seeded small-diamond descent transcribed from FFmpeg's `libavfilter/motion_estimation.c` (`ff_me_search_epzs`/`_ds`); predictors are the zero MV, the previous frame's global-motion MV, and the collocated previous-frame block MV (a persistent MV SSBO ping-ponged frame to frame). Value is **encode speed, not quality** — the MV field is a hint for an encoder's external motion search (e.g. NVENC `NV_ENC_EXTERNAL_ME_HINT`); the NVENC ME-hint consumer is a gated follow-up (ADR-0114 Tier 3), and it deliberately does **not** feed the denoiser (raw-pixel block-match is noise-limited, ADR-0113). AVOptions: `bsize` (default 16), `search` (default 24), `meta` (default on). No interop ABI bump — `PEL_SEC_MOTION` was reserved at ABI 1.0. Reference shader `libpelorus/shaders/pelorus_mc.comp` kept in lockstep. Shipped as `ffmpeg-patches/0007-add-vf_pelorus_mc_vulkan.patch`. ADR-0116, ADR-0113, ADR-0114.
