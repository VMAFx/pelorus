- **SVT-AV1 ROI steering** (`ffmpeg-patches/files/svtav1-pelorus-roi.patch`, `-pelorus_roi 1` on `libsvtav1` / `av1_svt`): stock `libsvtav1` consumes **no** ROI side data at all — the flagship modern AV1 software encoder had zero Pelorus steering. This patch consumes the same `AV_FRAME_DATA_REGIONS_OF_INTEREST` side data the merged `vf_pelorus_analyze roi=1` emits and maps it onto SVT-AV1's native per-64×64-superblock ROI segment map (`SvtAv1RoiMapEvt`): each frame's regions are rasterized onto the superblock grid and quantised into ≤ 8 AV1 segments, each carrying a `seg_qp` qindex delta (negative = more bits, matching `qoffset`), attached per frame via the `ROI_MAP_EVENT` private-data node with `enable_roi_map` flipped on at init. Per-frame events are heap-owned and freed at close (the library keeps the bare pointer live across its async lookahead). Compile-gated by `SVT_AV1_CHECK_VERSION(1, 6, 0)` (the release that introduced the ROI-map ABI) — older SVT-AV1 warns once and passes through; `enable_roi_map` is RC-mode agnostic but constant-quality (`-crf`/`-qp`) is recommended (the encoder's own AQ can override the map); zero behaviour change when off (default). `generate.sh` + `series.txt` extended to carry this libavcodec edit as patch 0012. **Verified on hardware** (real ffmpeg n8.1.1 + full stack, `--enable-libsvtav1`, SvtAv1Enc 4.1.0, RTX 4090 Vulkan): `analyze roi=1 → libsvtav1 -pelorus_roi 1` runs without crashing, the ROI is honored (bitstream + bitrate change only with the option on; engagement logged), banding drops modestly (CAMBI −1.5 % at CRF 35 / −0.5 % at CRF 45 on a synthetic localized-banding clip — an honest, modest gain, far below the NVENC −41 % on its harder source; a real-content BD-rate run is a follow-up). Film grain (`fgs_table`) deferred — SVT-AV1 exposes only a whole-stream init-time pointer, no per-frame film-grain channel (`FILM_GRAIN_PARAM` is commented out in the public header). ADR-0121.
