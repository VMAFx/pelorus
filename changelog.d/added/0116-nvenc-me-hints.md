- **NVENC external ME hints** (`ffmpeg-patches/files/nvenc-pelorus-me-hints.patch`, `-pelorus_me_hints 1`, H.264/HEVC): the gated Tier-3 consumer of the `vf_pelorus_mc_vulkan` producer (ADR-0116). Stock `nvenc.c` always runs the ASIC's own motion search and cannot be seeded; this patch reads the per-block MV field the producer emits as the `PEL_SEC_MOTION` interop section and feeds it to NVENC's external-ME-hint input (`enableExternalMEHints` + `maxMEHintCountsPerBlock` at init, per-frame `NV_ENC_PIC_PARAMS::meExternalHints` + `meHintCountsPerBlock`). One L0 candidate per 16×16 block (the SDK external-hint granularity for AVC/HEVC); the producer's `bsize` grid is resampled onto the 16×16 grid by center-block sampling; vectors are clamped to NVENC's hint bitfields (mvx S12, mvy S10). Value is **encode SPEED, not quality** — the ASIC seeds/short-circuits its own search. Default OFF (zero behaviour change); compile-gated by a new `NVENC_HAVE_EXTERNAL_ME_HINTS` macro (SDK 8.1+); capability runtime-probed (`NV_ENC_CAPS_SUPPORT_MEONLY_MODE`) with graceful degrade + one-shot warn; AV1 skipped (different per-superblock hint struct); the blob is parsed inline (no libavcodec→libpelorus link). **No speedup number ships** — the MV field is wired into NVENC's external-ME input and verified to compile against the ffnvcodec headers and replay onto pristine n8.1.1; the on-hardware encode-speed A/B is a documented follow-up. `generate.sh` extended to carry this libavcodec edit as patch 0008. ADR-0116, ADR-0114 Tier 3.
