So the proven `vf_pelorus_analyze roi=1` banding map now steers **Intel-hardware** bit allocation at full per-cell fidelity — mirroring the merged NVENC `qpDeltaMap` patch (ADR-0114 Tier 1). Portable by construction: 16×16 block size is the SDK-documented MBQP alignment for AVC and HEVC (not a GPU-specific coding-tree size); `EnableMBQP` is runtime-capability-probed (CQP rate control only — non-CQP warns once and passes through) and compile-gated by `QSV_HAVE_MBQP` (oneVPL/MSDK ≥ 1.13); no device-node/driver/entrypoint assumptions; zero behaviour change when off (default). `generate.sh` extended to carry this libavcodec edit as patch 0005.
