- **`pelorus_fgs` bitstream filter** (`ffmpeg-patches/files/h265_pelorus_fgs_bsf.c` → `libavcodec/bsf/pelorus_fgs.c`): the HEVC/H.274 leg of the film-grain round-trip the grain estimator (ADR-0115) started. Inserts an ITU-T H.274 Film Grain Characteristics (FGC) SEI message (HEVC payload type 19) as a prefix NAL into each HEVC access unit, so a conforming decoder re-synthesizes grain that was removed before encode (e.g. by `vf_pelorus_denoise_vulkan`). Stock FFmpeg HEVC encoders do not carry film-grain frame side data into the bitstream — the AV1 leg already round-trips through the native `AV_FRAME_DATA_FILM_GRAIN_PARAMS` channel, but HEVC had no automatic path — so this BSF carries the parameters into the elementary stream itself. Built on the coded-bitstream (CBS) generic BSF framework (`cbs_h265`), exactly like `hevc_metadata`: each access unit is parsed, one FGC SEI is added via `ff_cbs_sei_add_message()`, and CBS reassembles the unit. **Honest scope**: this is the static-model leg — the H.274 model is supplied via AVOptions (`model_id`, `blending_mode`, `log2_scale`, `components`, `intensity_low`/`high`, `scale_y`, `scale_c`, `persistence`, `skip_existing`), the canonical FFmpeg metadata-BSF contract; a BSF works on `AVPacket`s and no stock encoder forwards the estimator's per-frame grain side data onto the coded packet, so per-frame estimate→SEI plumbing through an arbitrary HEVC encoder is not expressible in n8.1.1. The producer's H.274 scalars (`h274_model_id`/`h274_blending_mode`/`h274_log2_scale`) and per-band RMS scaling map directly onto these options (mapping recipe in `docs/usage/grain-fgs-bsf.md`). Opt-in and no-op by construction (passes through unless a component is selected; `skip_existing` avoids double-stamping); does not link `libpelorus`. Verified end to end: SEI inserted into a libx265 stream and parsed back with `trace_headers` showing the exact configured values. Shipped as `ffmpeg-patches/0008-add-pelorus_fgs_bsf.patch`. ADR-0117, ADR-0115.
