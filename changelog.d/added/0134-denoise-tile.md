- **`vf_pelorus_denoise_vulkan`** gains an opt-in `tile` option (default off) that caches the current-frame NLM spatial search window in shared memory before the scan ([ADR-0134](docs/adr/0134-denoise-shared-mem-tile.md)). The kernel is fetch-bound — it re-reads an ~81-pixel window ~9× per output pixel — so `tile=1` is a large throughput win on bandwidth-limited GPUs (**+65.7%, 2.9×** on an Intel Arc A380), neutral on AMD RADV, and a small loss on the cache-rich RTX 4090 (its L2 already absorbs the redundant fetches), hence the flagship-first default. Output is **bit-identical** (SSIM 1.000000) to the direct-fetch path. Docs: `docs/metrics/denoise.md`, `docs/development/bench-results.md` v0.11.
